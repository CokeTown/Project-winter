# 2026-07-22 (수) 작업 일지

## 항공뷰 지도 설계 (디렉터 오더 — 구현 아님)

디렉터: 절차 생성 지도 이미지를 항공뷰 모델링으로 대체 + 노드 클릭 시 근접 환경 표시(레퍼런스: 다크 항공 지도 → 아이소 근접 디오라마). "내가 어딜 정확히 탐사하는지 알 수 있게." 필요한 것·가능 여부·파일 설계·자료구조 설계 요청.

- **현행 실측**: 지도=비오메 캔버스 dataURL(`mapBiomeDataUrl`, 서부/동부 분기) + `MAP_MARKERS` % 좌표 DOM 핀 — 배경과 노드가 **서로 다른 좌표계**라 "% 수작업 튜닝"(코드 주석 자인)이 구조적 부채.
- **설계서**: `design/systems/AERIAL-MAP.md` — 3안 비교(A 풀 라이브 3D=보류/B 베이크 파이프라인=1차 채택/C 근접만 라이브=2차 승격). 판정 근거 = 신규 엔진 기능이 아니라 **기존 자산 재배치**(오프스크린 하네스·셸터 buildEnv 16종·ruin-city 프리팹·#206 맨해튼 격자).
- **자료구조 핵심**: `mapnodes.js` = 도시 **월드좌표가 정본**(SSOT), 이미지 % 좌표는 베이크 도구가 항공 카메라 투영으로 **자동 산출**(`mapnodes.gen.json`) — 지형 바뀌면 재베이크만으로 마커 동행, 수작업 소멸. 뷰 상태 머신 city↔focus 2단.
- **파일 설계**: `ui/mapview.js`(openMapModal 추출 — 신규 기능을 God Object에 얹지 않음, #73 계승) · `render/dioramas.js`(C단계, `live:'env:rooftop'` Strategy) · `tools/bake-aerial.mjs`(시드 고정, icon-semiotic 계보) · `mapBackdrop(city)` Facade(베이크 부재 시 현행 캔버스 **폴백 강등** — 삭제 아님).
- **카메라 규약**: 전도·근접·인게임 전부 동일 아이소 각(골든 yaw0.6/pitch0.42 계열) — "줌인했다"는 착각은 시점 연속성에서 나온다(레퍼런스 사진1→2의 체험 핵심).
- **검증 설계**: 베이크 결정론(같은 입력=같은 PNG) · 골든 2씬 추가 · 좌표 정합 배터리 게이트. ⚠️ **P2 라이브 디오라마는 #212(lodge 플레이키) 함정 지대** — REFACTOR-GUIDE P1(렌더 컨텍스트) 선행 강권, 정적 베이크 단계는 리스크 0이라 계단 순서가 안전.
- **결정 대기 5건〔D〕**: 밤 전도 고정 1장(권고) · 근접 최종형(베이크vs라이브) · 줌 2단(권고) · 데모 포함(권고) · 지역 디오라마 아트 방향.
