### 1차 구현 (`ui-hud`, ui-semiotic에서 분기)

- **E-바**: 좌상단 2×4 그리드(스펙이 명시 제거 대상) → 하단 중앙 명령 바. 주 5(탐험·제작·취침·청소·이주) + 구분선 + 보조 3(지식·옷장·도움, .sec 축소). 버튼 id·배선 불변(이동만). title/ui-hidden/wallpaper 모드 숨김 배선. 팁 쪽지 bottom 22→86px(겹침 회피).
- **게이지 심각도**: `gaugeSev()` 공용(0~25 위험/26~50 주의/51~ 정상, 스펙 임계로 45→50 이동) — HUD `.g-fill`·도킹 LED·PDA `.pbar` 3면 동일 토큰(NFR-004). 색=심각도(그린/앰버/적), 게이지별 고유색 폐지. 수치 병기 복원 + 상태 문자(주의/위험) — 로케일 3언어 6파일 `g.state.warn/crit` 신설(2196키). 점멸 1.3s 주기(≤0.8Hz 스펙 상한, 기존 0.85s 위반이었음).
- **A패널 행 스택**: 게이지 3열 그리드 → 행(라벨|세그먼트 바|수치) — 스펙 6.1 문법. 수치는 게이지 루트에 append(행 순서 확보).
- **검증**: 하네스 캡처 3회전 — 1차에서 도움 버튼 랩 이탈+쪽지/캠버튼 겹침 검거 → 슬림화+쪽지 상향으로 해소. 심각도 강제 캡처(기력 15)로 적색 바+위험 문자+수치 동시 발현 확인(ACC-04). 갈증 40 주입은 오토드링크가 85로 복구 — 시뮬 살아있음의 방증. i18n 2196 무결 · 하드코드 신규 0.

- **이동**: `#action-bar` 하단 중앙 → 상단. 뷰포트 중앙 정렬은 1280에서 시계와 71px 겹침(브라우저 rect 실측) → **좌 HUD·우 시계 사이 좌측 고정 밴드**(left:336·max-width:440)로 전환. 시계 좌측 가장자리가 뷰포트 폭과 무관하게 ~783px 고정(줌·레이아웃 앵커)이라 우측 앵커 대신 좌측 고정이 정답.
- **검증(브라우저 rect)**: 1280→바 336–763·HUD끝 328·시계 783 겹침0 / 1920→바 336–763 동일 겹침0 / 390 모바일→상단은 HUD와 세로 겹침 → **모바일은 하단 유지**(엄지 도달성 우위, 디렉터 "상단으로"는 와이드 대상). 일시정지 표시 62px 하강·쪽지 하단 복원.
- **육안(1280 오프스크린)**: 명령 바 상단 밴드 정착, 게이지 심각도(기력 위험 적색) 동시 확인.
- **게이트**: i18n 2196 무결·하드코드 신규 0·배터리 102/102·모달 DOM 7/7.
- **⚠️ 골든 lodge 플레이키(HUD 무관·기존)**: 재고정 직후 check1=0.041%(완벽)인데 check2=lodge 4.6%/8%. **두 체크는 별개 프로세스** — lodge의 옥외 요소(관측소·케이블카 비동기 빌드 또는 반투명 z-정렬)가 오프스크린 SwiftShader 크로스프로세스 순서에 따라 양자 상태로 뒤집힘 = [[#212]] 미해결분과 동종. **HUD 변경 무죄 근거**: diff가 블록 8%의 덩어리 3D 오브젝트(상단 얇은 바 아님) + 직전 두 패스는 우연히 같은 상태로 캡처돼 통과했을 뿐. 관측소/케이블카는 전부 Lambert(불투명)·창 스카이는 MeshBasicMaterial(불투명)·성에는 opacity0 — 가시 반투명 부재라 정착 타이밍 유력. **보고: 16/17 안정 + lodge 1건 기존 플레이키. 원인 규명은 골든 결정론 후속(#212 계열)으로 이관, HUD 커밋을 막지 않음.** 커밋된 baseline은 state A(check1 확인).
