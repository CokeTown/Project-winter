## §1. 진단 — v6이 밋밋한 이유 (수리 대상)

| # | 문제 | 증상 |
|---|---|---|
| 1 | **균일 페이싱** | 모든 컷이 2~4초. 숨 고르기도 폭발도 없다 — 음악만 아치가 있고 편집은 평평 |
| 2 | **단일 샷 문법** | 거의 전부 아이소 와이드 + 드리프트. 클로즈업 2개(난로·고양이)뿐 |
| 3 | **대비 셋업 부재** | 혹독함을 먼저 3초 보여주지 않아 온기가 "당연"해짐. 코지의 감정 공식 = 추위×온기의 낙차 |
| 4 | **감각 사운드 부재** | 라이터·장작뿐. 골골·문·눈 밟기·장소 앰비언스 0 — 힐링은 청각이 절반인데 음악 혼자 일함 |
| 5 | **UI 비중 과다** | 탐험 8.2s + 해금 7.1s = 15.3s(32%)가 UI 화면. 슬라이스 전달엔 좋지만 무드를 끊음 |
| 6 | **피크 부재** | "오!" 하는 단일 순간이 없음. 전부 같은 온도의 나열 |

**결론: 기능 나열(checklist)이 아니라 감정 곡선(arc) 위에 슬라이스를 얹어야 한다.**

## §2. 트레일러 A 최초 기획 — 후킹용 「그래도 오늘 밤은 따뜻하다」 (~52s)

**목표 감정**: "아 편안하고 힐링되네, 한번 사볼까?"
**프레임**: 기능 나열 폐기 → **생존자의 하루**. 아침에 나가고, 오후에 살림을 늘리고, 밤에 함께 쉰다.
**감정 곡선**: 혹독(파랑) → 첫 불씨 → 살림의 리듬 → 함께함(피크) → 밤의 고요 → CTA
**3대 수리 레버**: ① 컷마다 디제틱 사운드 1개 ② 타이트↔와이드 교차 ③ UI 총량 ≤ 6초

## §3. 트레일러 B 최초 기획 — 세계관용 「아홉 번째 겨울」 (~60s)

**목표 감정**: "이 세계엔 이야기가 있구나. 이 겨울 끝에 뭐가 있지?"
**프레임**: A가 '삶'이라면 B는 '세계와 물음'. 라디오 텍스처 + 문서 조각 + 침묵의 도시 + 응답의 약속.
**구성**: 스태틱→주파수(오프닝 후킹) / 황폐 도시 와이드들 / 기밀 문서 3장 인서트 / 봉쇄선·낙진 /
방호복 발자국(만나지 못한 타인) / 무전 송출→응답 불빛(여기선 점이 정답 — 서사가 '응답') /
Winter 9 타이틀 / 동부 실루엣 티저(2.0) / 로고.
**음악**: Gloomy 계열 + radio_static/broadcast 사운드 디자인. 자막 2~3장만(DungGeunMo).
**금지**: 침묵 루트 노출 절대 불가(#169 유출 스탠스). 힌트도 안 됨.
**용도**: 스토어 2번 슬롯 + 출시 후 공지/2.0 예고 재활용.

## §5. 확장 (루틴 산출 — 채택분)

1. **장소 앰비언스 스티치** (핵심): 몽타주 컷마다 그 장소의 소리 0.5레이어 — 컨테이너=amb_wind, 지하철=울림(잔향), 온실=amb_spring_birds, 오두막=amb_fire, 요트=물결, 로지=amb_winter_gale. 파일 전부 보유(public/SFX). 몽타주가 '기능 나열'에서 '장소 여행'으로 바뀐다.
2. **골골 사운드**: purr.mp3 보유 — 쓰다듬기 피크에 필수. 이게 힐링의 방아쇠.
3. **창가 성에 클로즈업**: frostState 훅 존재 — 성에 낀 유리 안팎 대비 컷 (코지의 본질 1장).
4. **도시 소등 와이드 엔딩**: 줌 최소(0.2~0.35)로 빠지며 어두운 도시에 우리 창만 밝게 — "이 온기는 내가 만든 것" 마침표.
5. **아침 출발을 몸짓으로**: 문 열림(door.ogg)+눈 밟기(steps_snow)+아바타 퇴장 — UI 없이 '나간다' 전달.
6. B **주파수 다이얼 오프닝**: 검정 화면 + 스태틱이 방송 조각들을 스치는 사운드만 3초 — 장르(포스트아폭 무전)를 귀로 먼저.
7. B **문서 인서트**: showMemoPage/showTruthPage 실물 종이 렌더 — 판데믹 기원→소각 결정→관측 프로그램 순 3장, 장당 2초.

**기각**: 도색 컷(A에 과밀), 계절 4연변(길이 초과, B의 도시 와이드로 대체), 침묵 힌트(스탠스 위반), 내레이션(톤 리스크+비용).

## §6. 반박/재반박

| # | 반박 | 재반박 → 결정 |
|---|---|---|
| R1 | UI를 빼면 "게임이 뭔지" 전달 약화 — 디렉터가 v4에서 UI를 요구했다 | 오더의 본질은 "보내는 게 보여야". 몸짓(문 열고 나감→정산 개봉 1.5s)으로 충족하고, UI는 **정산 개봉 + 해금 지도 압축본(4s)만** 잔류. UI 총량 15.3s→5.5s. 슬라이스 전달과 무드 양립 |
| R2 | 2편 = 비용 2배, 넥페 전 시간 부족 | 파이프라인이 스펙 기반이라 A는 v6 자산 ~70% 재사용(반나절), B는 신규 7~8클립(1일). 총 1.5일 — 감내 가능 |
| R3 | B의 어두운 톤이 스토어 첫인상을 해침 | 슬롯 순서로 해결: 스토어 1번=A, 2번=B. Steam 다중 트레일러 지원. B는 공지·2.0 예고로 수명 연장 |
| R4 | 디제틱 사운드가 유튜브/스팀 압축에서 뭉개짐 | 사운드 이벤트를 음악 저점(물방울 구간)에 배치 + 최종 -14 LUFS 라우드니스 정규화 패스 추가 |
| R5 | 몽타주·해금을 남기면 여전히 나열 아닌가 | 앞뒤를 감정 프레임(혹독→온기 / 피크→고요)으로 감싸고 컷마다 장소 소리를 얹으면 나열이 아니라 '가능성의 여행'으로 읽힘. 해금은 12비트→8비트로 압축해 리듬 유지 |
| R6 | 하루 프레임에서 몽타주(다른 집들)가 논리 파열 | 하루가 아니라 **'하루의 형식을 빌린 일생'**으로 — 몽타주 직전 컷을 "짐 싸는 손"(이주 모티프) 대신 지도 해금으로 연결해 "넓어진 만큼 집이 늘었다"로 정당화. 편집 논리는 음악 드랍이 캐리 |
