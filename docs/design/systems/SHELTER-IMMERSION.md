# 셸터 몰입 — "통이 아니라 장소로"

> **디렉터 방향 확정 2026-07-24.** 게임 정체성 점검에서 출발: cozy 루프는 작동하나, 각 셸터의
> 스토리·세계관이 빈약해 몰입 여지가 적다. 해법은 시스템 추가가 아니라 **밀도**다.
> 정체성 = 코지-퍼스트(캐논 「따뜻한 고독, 응답하는 불빛」 / 「살림을 판다, 서바이벌 아님」)는 유지.

---

## 0. 한 줄 진단 (왜 이 문서인가)

- **루프는 OK**: 뭐하지 → 탐험 → 자원 → 꾸며짐 → 부족 → 탐험, + 인카운터가 "세상이 다 죽진 않았구나"를 줌. 거처 옮기며 내 셸터 만들기. → 코지로 작동.
- **구멍**: 셸터마다 **스토리·세계관 빈약** → 몰입할 여지가 적다. 셸터가 "가구 얹는 빈 통"이다.
- **"목적 없음"은 반쯤 QA 아티팩트**: 버그 잡으며 플레이하면 아치(아홉 겨울·응답 불빛·엔딩)가 체감 안 됨. 실플레이어는 더 느낌 → 덜 급함(척추 연출 강화는 별개 트랙).
- **아트 atmosphere 자체는 좋다** — 바닥이 튼튼하니 갈아엎지 말고 밀도만 얹는다.

**통합 프레임**: 네 갈래 전부 = **"셸터를 장소로."** 누가 살았나(A 스토리) · 어디 서 있나(B 지형) · 밖은 어떤 세상인가(C 인카운터 분위기) · 그걸 무엇으로 굴리나(D 시스템 고도화).

---

## 1. 네 갈래 (비전)

### A. 셸터 스토리 — "누가 살았던 자리"
각 셸터에 **떠난 사람의 흔적**을 깐다. 도착 비트(여기 누가 있었구나) → 청소·정착하며 나오는 유품/메모 → 조금씩 맞춰지는 짧은 실. 꾸미는 행위가 *남의 삶 위에 내 삶을 얹는 것*이 된다.
- 캐논 정합: 라디오로 먼 불빛이 답하듯, **내 집 자체가 한 사람의 삶을 기억**한다. 코지 톤(쓸쓸한 온기) 유지, 생존 압박 0 증가.
- **재활용 인프라**(신규 시스템 아님): 메모(`dropMemo`/`showMemoPage`) · 「생존의 흔적」 프롭(#190) · 발견 컷 비네트(`vignettes.js`) · 스케치 액자(#134) · 지역 2겹화 겉/속(#167).

### B. 셸터 비주얼 다양성 — "평지에 건물" 탈피 (자연 셸터)
자연 셸터도 평면 위 박스라 지형 드라마가 없다. 각 셸터에 **풍경**을 준다: 지형 고저 + 자연 둘러쌈 + 원경 레이어(전경 바위·나무 → 셸터 → 배경 산·바다·숲선).
- 오두막=완만한 비탈 숲 빈터(전나무·언 개울·장작) · 등대=바다로 튀어나온 바위곶(수평선=바다·파도) · 스키로지=설사면+뒤 산·수목한계선 · 온실=야생화된 들판·죽은 억새.
- **재활용**: 도심 잠식(#71)·계절 식생 파이프를 자연 지형으로 확장. 신규 엔진 아님.

### C. 탐험 인카운터 — 두 갈래 (디렉터 지목)
- **CRT / 카세트 퓨처리즘 발견물**: 탐험 중 *아직 돌아가는 기계*를 만남(보안 콘솔·응답기·루프 도는 CRT·메인프레임). **MU/TH/ER 터미널 문법으로 "조작"**하는 인카운터. 월드빌딩 + 전리품. 관측 단말(#217)·PDA 터미널 미감 그대로 확장.
- **상호작용 인카운터 (VN·촉각)**: "상자 여는" 손맛(#133 DDD-1)을 넓혀 — 금고(발견 조합 입력)·라디오 다이얼 맞추기(신호 잡기)·갈래 있는 장면(문 앞 낯선 이·라디오 너머 생존자). 텍스트 카드보다 한 겹 선택·손맛.

### D. 현행 시스템 고도화 (신규보다 깊이)
- **라디오** = 테제 시스템인데 지금 방송→응답 불빛에서 멈춤. → 주파수 튜닝·다른 생존자 포착·방송 왕복으로 관계가 쌓이는 상호작용 시스템. (속편 떡밥 「낯선 교신」이 여기 착지.)
- **탐험** = 방치 파견+정산. → 접근법(빠르게/신중히)·지역 숙련(#123 기존)이 결과에 더 표현.
- **쾌적/지식** = 셸터 스토리와 엮어 "기억하는 집" 보너스 시너지.

## 2. 파일럿 스펙 (지금 착수: A 옥탑 + C CRT)

> Next Fest(10월) 감안 — 전 16셸터 말고 **데모 주인공 옥탑**으로 밀도를 증명. 옥탑 스토리 +
> CRT 인카운터 1종이 서로 물려(누가 살았고 → 밖에서 그 기계를 만나고 → 라디오로 답이 온다) 효과가 즉시 보인다.

### 2-A. 옥탑 스토리 — 「먼저 부른 사람」

**서사(제안 — 디렉터 승인/윤문 대상)**: 옥탑의 이전 거주자는 **어둠 속으로 신호를 보내려 여기 올라온 첫 번째 사람**이다. 라디오가 그의 것이었고, 밤마다 불렀다. 지금은 없다(떠났는지·죽었는지 모호하게). 남은 것: 라디오, 접이의자, 담요, 난간에 그은 밤의 금들, 그리고 로그.
- **왜 강한가**: 게임 코어(라디오/응답)를 셸터 스토리로 국소화. **네 라디오 송신 = 누군가의 파수를 잇는 것**이 된다. 네게 답하는 먼 불빛도 저마다의 파수를 잇는 이들 — 테제가 집 안에서 한 번 더 울린다.

**비트 3단**(전부 기존 인프라):
1. **도착**(옥탑 최초 정착): 짧은 메모 발화 — 옥상엔 이미 라디오·접이의자·난간의 금들이 있다. "여기서 누군가 오래 불렀구나."
2. **유품**(청소·정착 진행에 따라 1~2개씩): 로그 페이지가 떠오른다(showMemoPage 재활용). 부르던 밤들, 곱은 손, 사그라듦.
3. **실 마감**(마지막 로그): 그가 **딱 한 번 답을 받은 밤** — 그 뒤는 백지. "누군가 들었다. 그거면 됐다." 너는 그 라디오를 이어받는다.

**제안 문안(KO 초안 — EN/JA는 구현 시 3언어)**:
- `shelterlore.rooftop.arrive`: "옥상에 라디오가 있었다. 누가 두고 갔다. 접이의자 하나, 담요, 난간에 그어진 금들 — 밤을 세는 자국. 여기서 누군가 오래 불렀구나."
- `shelterlore.rooftop.log1`: "사흘째. 신호를 보낸다. 아무도 없다. 그래도 매일 밤 올라온다."
- `shelterlore.rooftop.log2`: "스무 밤째. 손이 곱아 다이얼이 겨우 돌아간다. 그래도 돌린다."
- `shelterlore.rooftop.log3`: "오늘 — 저 멀리 창 하나가 잠깐 켜졌다. 누군가 들었다. 그거면 됐다." (뒤는 백지)

**구현 노선**(신규 시스템 최소):
- 셸터별 lore 테이블 `data/shelterlore.js` 신설(`{ shelterId: { arrive, beats:[...] } }`). 술어는 `state.shelterLore[shelterId]` 진행도.
- 도착 비트: `loadShelter` 최초 진입(해당 셸터 lore 미개봉) 시 arrive 메모 1회.
- 유품: 정착 진행(정착도/청소/일수 게이트)에 beats[i]를 순차 개봉 → showMemoPage.
- 옥탑 씬에 **정적 프롭**(접이의자·담요·난간 금) 배치 — 이미 있는 옥탑 빌더에 소품 추가(골든 재핀).
- 세이브 왕복: `state.shelterLore` 직렬화(왕복 게이트).

### 2-C. CRT 인카운터 — 「아직 돌아가는 단말」

**컨셉**: 탐험 중 낮은 확률로 *백업 전원으로 아직 켜진 CRT 콘솔*을 발견. MU/TH/ER 터미널 문법(인광 그린·스캔라인·`//` 헤더·점멸 커서)으로 부팅→로그인 프롬프트→선택.
- **읽기**(월드빌딩): 마지막 시스템 로그 — 여기서 무슨 일이 있었나(대피·역병·정전).
- **캐시 인출**(전리품): 남은 데이터를 뽑는다 — 도면/자원/메모 롤.
- **전원 내리기**(감정 비트): 그냥 꺼준다. 작은 안식.

**제안 스크립트(KO 초안)**:
- 부팅: `// LINK ESTABLISHED — 단말 온라인 (백업 전원 잔량 7%)`
- 본문: "사무실 잔해 속, CRT 하나가 아직 웅웅거린다. 커서가 깜빡인다. 마지막 사용자가 로그아웃하지 않았다."
- 선택: ①마지막 로그 읽기(월드빌딩 텍스트) ②캐시 인출(전리품 롤) ③전원 내리기(무보상·안식 노트)

**구현 노선**:
- `data/events.js`에 인카운터 1종 추가(`crt_terminal`) — 기존 EVENTS 구조(`titleId/textId/choices[{run()}]`) 재활용.
- 표현은 이벤트 카드에 **터미널 스킨 변주**(설정창·관측 단말 CSS 재활용) 또는 경량 비네트. 1차는 카드+터미널 스킨으로 최소.
- 탐험 인카운터 풀에 게이트(도심·상업지구 등 실내 지역 한정, 낮은 확률).

## 3. 순서 · 열린 결정

**착수 순서**(이번): ① 옥탑 스토리(도착+유품+실) → ② CRT 인카운터 1종. 그 다음 B(오두막 지형 파일럿) → D(라디오 상호작용 씨앗).

**열린 결정〔D〕**:
1. **옥탑 서사 승인/윤문** — 「먼저 부른 사람」 방향 + 위 로그 문안. (세계관 정합은 Fable가 캐논 대조했으나 최종 문안은 디렉터 몫.)
2. **CRT 인카운터 톤** — 월드빌딩을 얼마나 무겁게(역병/대피의 구체) vs 담백하게. 군사톤 금지(카세트 퓨처리즘 규약).
3. **B 지형 파일럿 대상** — 오두막 vs 등대(둘 다 자연, 등대가 지형 드라마 큼).

**검증 규약**(구현 시): 신규 문자열 3언어×6파일 · 배터리 102/102 · 옥탑 프롭 추가 시 골든 rooftop 계열 재핀 · 인카운터는 코어 배터리에 술어 테스트 · 세이브 왕복(shelterLore).
