## 1. 정본 규격 (신규/수정 UI가 따르는 계약)

- **픽셀 그리드**: 게임은 `innerWidth / opts.pixel`로 렌더(1텍셀 = `opts.pixel` px). UI 텍스처도 **동일 텍셀 크기**를 목표. 픽셀본은 저해상으로 굽고 `image-rendering: pixelated`로 확대(다운스케일 금지 — 도트가 뭉갬).
- **팔레트**: 공용 한정 팔레트로 스냅. 정본 도구 `tools/icon-pixelize.mjs`(grid24·K32·베이어). 아이콘 세트는 **세트 전체 1회 median-cut**(낱장 팔레트 = 불균질 = AI티). 고채도 액센트(앰버·적색 LED·인광)는 전용 슬롯 보존.
- **폰트**: `DungGeunMo` 전면. 명시 sans-serif/시스템폰트 금지, `font-family: inherit` 강제. 크리스프 사이즈(네이티브 그리드 배수)는 **실화면 검증으로 확정**.
- **보더**: 2px 하드 픽셀 보더. `border-radius ≤ 2px`(LCD/종이 내부는 0). **1px 이하 AA 보더 금지**(창=도트, 버튼=AA 로 쪼개지면 안 됨).
- **image-rendering**: 모든 픽셀아트 표면에 `pixelated`. 스무스가 정당한 곳(초소형 다운스케일 등)만 예외 + 이유 주석.
- **선명도 위계**: 1차 정보(자원·상태)=선명·채도. 2차(장식·배경)=디밈·저채도. 채도 튀는 아이콘이 무드 깨면 팔레트로 낮춘다.
- **텍스트 안전**: 아이콘→텍스트 순, `keep-all`+말줄임/줄바꿈 규칙, 컨테이너 밖 삐짐 0.

## 5. 아이콘 언어 — 「도트 이후」 (2026-07-20 피드백 · **디렉터 GO — 세미오틱 글리프안 채택, `ui-semiotic` 브랜치 구현 중**)

> **동인**: 커뮤니티 피드백 2건(2026-07-20). ① *"인게임이 픽셀이긴 해도 해상도가 상당히 높은데 UI 픽셀이 더 단계가 적어 보여서 도트일 필요가 있나 싶은 경계감… 특히 핸드폰(PDA)은 혼자 색상도 리얼리티하게 많이 들어가서 붕 뜸."* ② *"픽셀에 대해 모르는 것 같으니 그냥 픽셀을 하지 마라."* — 공통 결론은 "아이콘을 더 잘 그려라"가 아니라 **"어설픈 도트는 신뢰를 깎는 마이너스 자산"**.

### 5.1 문제 규정

- 인게임 렌더(1텍셀=`opts.pixel`px, 고밀도)와 UI 도트(32그리드 확대, 저밀도)의 **텍셀 단계가 2계층으로 갈라져** 있다. §2-A(크기 7종 혼용)가 이걸 증폭. → 보는 사람은 "왜 UI만 더 굵은 도트냐"를 즉시 감지한다.
- icon-pixelize는 AI 래스터의 **다운스케일 근사**라, 도트 눈이 있는 검증자에게는 "손도트"로 읽히지 않는다. 텍셀을 통일해도(기존 P1) 이 리스크는 남는다.
- PDA·아이콘의 리얼컬러가 레트로 팔레트를 깬다(규율 2의 외부 재확인).

### 5.2 통찰 — 답은 컨셉 안에 있다

현 UI 컨셉 = 노스트로모 MU/TH/UR. **원전 화면 언어에는 도트 아이콘이 없다.** 원전의 그래픽 문법은 ①모노스페이스 대문자 타이포 ②세그먼트/바 ③1px 라인 스키매틱(배관도·평면도) ④**기하 픽토그램** — 론 콥(Ron Cobb)의 *Semiotic Standard*(노스트로모 선내 표지판 세트: 원·삼각·사각·호의 단색 조합) — 뿐이다. 즉 "도트가 아니라면 무엇인가"의 답은 이미 정해져 있다: **UI는 도트 그림을 버리고 발광 벡터 글리프로 간다.** UI가 "콘솔 스크린 속 발광 그래픽"이라는 별도의 다이제틱 재질이 되는 순간, 인게임 픽셀과의 텍셀 단계 비교 자체가 성립하지 않는다(문제 ①의 원천 소멸).

### 5.3 제안 규격 — MU/TH/UR 네이티브 2층 언어

| 층 | 원칙 | 적용처 |
|---|---|---|
| **1층 텍스트-퍼스트** | 아이콘 없이 라벨+수치+상태 LED. v1.9.10이 이미 이 방향(게이지=라벨+바, 크래프팅 탈아이콘) — 유지·확대 | 리스트, 제작, 게이지, 메뉴, 설정 |
| **2층 세미오틱 글리프** | 공간이 좁거나 스캔 속도가 필요한 곳만 단색 라인 글리프 | 도크, 지도 마커, 자원 칩, 상태 배지, 액션 버튼 |

**글리프 규격**: 24 뷰박스 · 2px 스트로크 · 기하 도형(원/삼각/사각/직선/호)만 · fill 최소 · 모서리 각(라운드 금지) · **단색 `currentColor`** — 색은 글리프가 아니라 상태가 결정(인광 그린=정상 / 앰버=경고 / 적=위급, 세그먼트 게이지와 동일 문법). 렌더는 inline SVG 또는 CSS mask → 테마색 상속 공짜, 어느 해상도·`uiz`에서도 crisp(크기 7종 문제도 함께 소멸).

**제작 방식**: AI 생성 아님(07-16 오더 준수) — 기하 조합이라 **코드로 그린다**(`tools/gen-icons.mjs` 전례 → `icon-semiotic.mjs`). 파라메트릭이라 수정·증설 재현 가능.

**PDA**: 리얼컬러 소거 → 인광 모노크롬(사진·지도 등 컬러 콘텐츠는 인광 듀오톤 처리). 터미널 언어의 발원지였으니 자연 회귀(문제 ① PDA 지목 해소).

### 5.4 대안 비교와 기존 플랜 관계
