# UI 디자인 토큰 정본 (AI 티 처방 2번)

> 2026-07-16 갤 피드백 "선 굵기/폰트크기/색/행간까지 디자인하듯 디테일하게" 대응.
> 원인 진단: 값을 그때그때 정해 폰트 15종(8~26px, 10.5px 포함)·라운드 9종(1~8px)·박스섀도 34곳이 누적 — 사람 디자이너는 3~4단계 스케일 안에서만 움직인다. 이 문서가 그 스케일의 정본이다.
> 규칙: **신규 CSS는 아래 토큰 밖의 리터럴 값 금지.** 기존 표면은 만질 때마다 토큰으로 수렴(#199 블록부터 적용 완료).

## 1. 색 — 기존 :root 6+2, 신규 hex 발명 금지

| 토큰 | 값 | 용도 |
|---|---|---|
| `--panel-bg` | rgba(24,21,17,.94) | 패널 바탕 (웜 차콜) |
| `--panel-border` | #4a4136 | 패널 테두리 |
| `--panel-edge` | #14110d | 외곽 프레임 |
| `--accent` | #e8a86c | 강조 (랜턴 앰버) |
| `--good` / `--bad` | #8fbb7a / #c96a5f | 상태 |
| `--text` / `--text-dim` | #d8d5cc / #9a9184 | 본문 / 보조 |
| `--lcd-ink` | #9fe8a0 | PDA·필드 터미널 포스퍼 그린 (LCD 안 전용) |
| `--lcd-dim` | #5a8f5e | LCD 보조선·비활성 |

게이지 고유색(허기 앰버·갈증 블루·기력 그린)은 데이터 색으로 예외 — 신규 추가는 이 문서에 등록 후.

## 2. 타이포 — 3단계 + 디스플레이 예외

| 토큰 | 값 | 용도 | 흡수 대상 |
|---|---|---|---|
| `--fs-s` | 10px | 캡션·태그·보조 수치 | 8, 9, 10.5, 11px |
| `--fs-m` | 12px | 본문·버튼·리스트 행 | 13px |
| `--fs-l` | 15px | 창 제목·섹션 헤더 | 14, 16, 17px |
| `--fs-display` | 26px | 시계 숫자 등 계기 디스플레이 | 18~26px 산발값 |

행간: 본문 `1.5` / 계기·숫자 `1.2`. 자간: 제목·라벨만 `0.5px`.

## 3. 형태

- 라운드: `--r: 3px` **단일**. 종이(수첩·팁)와 LCD 스크린 내부 요소는 0 (기물은 각지다).
- 선 굵기 2단: 내부 구분 `1px`, 외곽 프레임 `2px`. 3px+ 테두리 금지.
- 그림자: 패널 드롭 섀도 1규칙(`0 4px 12px rgba(0,0,0,.45)`)만. 요소 단위 box-shadow 남발 금지.
- 글로우: **화면당 활성 요소 1곳만** (현재: PDA 전원 LED). text-shadow는 LCD 포스퍼 블룸(`0 0 6px` lcd-ink 30%) 1규칙 외 금지.

## 4. 아이콘

- 전 UI 아이콘 = `tools/icon-pixelize.mjs` 규격(32그리드·공용 64색 median-cut·오더드 디더 밴드 0.28·×8 업스케일)을 통과한 진짜 도트만. 파이프라인: `node tools/icon-pixelize.mjs --grid 32 --k 48 --up 8 --out <dir> --src public/img/icons` 후 public 반영.
- AI 생성 원본 직행 금지(로고와 동일 병리 — 격자 없는 래스터). 신규 아이콘은 디렉터 도트 원본 우선.
- 기본 이모지는 UI 표면 금지(토스트 등 텍스트 채널은 잠정 허용 — 전환 대기 목록은 워크로그 §32).

## 5. 미결 (디렉터 선택)

- **A안 현행 LCD 다듬기 vs B안 플랫** — 시안 캡처 비교 후 확정 (211214 "원래 플랫이 낫다" 의견 참고). 확정 전 전 UI 스윕 보류, #199 블록은 토큰 수렴만 선반영.
