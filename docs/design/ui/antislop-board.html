상인 이미지 "AI 슬롭" 진단 ev_trader vs ev_cat 실증 대조 · 2026-07-14

피드백: "다른 건 분위기 맞는데 중간에 상인 이미지가 너무 AI 슬롭 같다." 같은 파이프라인·같은 styleAnchor인데 두 컷의 결과가 갈린 이유를 실제 에셋으로 비교.

한 줄: 상인이 유독 슬롭인 건 ① 세트의 다른 컷은 픽셀·디더 질감이 살아 "의도된 픽셀아트"로 읽히는데 상인만 그 질감이 날아가 매끈한 제네릭 페인터리가 됐고, ② 상인은 얼굴·손·복제된 냄비 더미가 있는 캐릭터 클로즈업이라 AI 최약점이 정면 노출됐기 때문. 뿌리는 프롬프트의 자기모순 "pixel art … painterly pixel scene".

슬롭

  • 픽셀·디더 질감 소실 → 매끈한 에어브러시 페인팅

통과

public/img/events/ev_cat.png
  • 픽셀 그리드·디더링 뚜렷 → 의도된 픽셀아트로 읽힘
  • 오브젝트 개별 분리, 반복 클로닝 없음
  • 환경 중심 컷, 고양이는 뒷모습(얼굴 없음)
  • 창 하나 앰버 + 나머지 냉색 대비 (균일 발광 아님)
상인 확대 — nearest 확대해도 픽셀 각이 없다. 매끈한 블렌딩·뭉갠 냄비. 픽셀아트가 아니라 페인팅.
고양이 확대 — 낙엽·바닥에 실제 픽셀·디더 그레인이 보인다. 세트가 공유해야 할 질감.

고치는 3레버

레버 A · 가장 확실진짜 픽셀화 후처리다운스케일 → 팔레트 양자화 → nearest 업스케일. 전 컷을 같은 픽셀 그리드·팔레트로 통일하면 "상인만 튐"이 구조적으로 사라짐. (게임 본편 복셀+픽셀 포스트와도 정합)
레버 B · 텔 숨기기인물 컷 문법대면 인물은 실루엣·후면·원경·후드그림자로 얼굴·손을 화면에서 뺀다. 반복 오브젝트는 few·distinct. 세계관("타인은 스치고 흐른다")과도 맞음.
레버 C · 일관성레퍼런스 먹이기잘 나온 ev_cat을 edit 엔드포인트 레퍼런스로 넣고 input_fidelity high → 세트 실측 질감이 전이돼 튀는 컷 원천 차단.

프롬프트 수정 핵심

styleAnchor에서 painterly 제거 → pixel art 단호하게 + 네거티브 "NOT smooth, NOT painterly, no HDR bloom, no cloned objects". 상인 프롬프트는 "stacked high with pots"(대량 반복) → "a FEW distinct, individually readable pots", 인물은 후면·후드그림자·프레임에서 작게. 전체 문안·후처리 코드·큐레이션 절차는 동봉 가이드 AI-ART-ANTISLOP.md §4·§6 참조.
