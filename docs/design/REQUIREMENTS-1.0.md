## 4. QA 프로세스 (사람과 하네스의 분업)

- **하네스[H]가 잡는 것**: rect/좌표, 수치, 상태 전이, 저장 왕복, 조건 발화, 회귀 — 릴리즈마다 전 스위트 재실행(ui/mode/conv/plaus/edit/confirm/notice/light+신규)
- **시뮬[S]이 잡는 것**: 곡선(자원/게이지), 확률 밴드, 장기 안정성
- **육안[V]이 잡는 것**: "이상해 보임"(포즈/비율/색/톤) — 스크린샷 갤러리를 코디네이터가 판정, 애매하면 디렉터 에스컬레이션
- **디렉터[E]가 잡는 것**: 실기기 체감(터치/성능/설치), 재미, 그리고 **찐빠** — 시트 트리아지 루프(feedback-loop-protocol)로 환류. 🔴+항상 = 즉시 핫픽스
- 원칙: **하네스로 잡을 수 있는 걸 사람이 잡았다면 그건 하네스의 버그** — 재발 방지로 해당 케이스를 스위트에 추가한다
