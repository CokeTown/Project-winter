## 2. 이미 잘 된 축 — 이 패턴을 복제한다

리팩터의 방향은 새로 발명할 게 아니라, **이 저장소에서 이미 배당금을 낸 3가지**를 확장하는 것이다.

| 사례 | 패턴 | 배당금(실증) |
|---|---|---|
| `icon(name)` 단일 관문 | **Facade / 간접층** | 도트 PNG 72종 → 세미오틱 글리프 전환 시 **호출부 261곳이 아니라 함수 1개**만 수정 (2026-07-20) |
| `i18n.pick()` 조회 병목 훅 | **Interception Point** | 로케일 3언어에 박힌 인라인 아이콘 261건을 **JSON 무수정**으로 일괄 치환 |
| `gaugeSev(v)` 술어 추출 | **Single Source of Truth** | 심각도 임계(25/50)가 HUD·도크 LED·PDA 3곳에 리터럴로 흩어져 드리프트하던 것을 술어 1개로 수렴 (2026-07-21) |

**교훈**: 넓게 퍼진 변경 비용은 **관문(seam)이 있느냐**로 결정된다. 관문이 있으면 1곳, 없으면 N곳. 아래 개선 포인트는 전부 "없는 관문을 만드는 일"이다.

## 3. 개선 포인트 (우선순위)

각 항목: **증상 → 측정 → 패턴 → 처방 → 검증 → 위험**.

### P1. 렌더 모드 플래그 관통 → 렌더 컨텍스트 객체
- **증상**: `_golden`(+`_goldenDt`/`_goldenAcc`/`_goldenKeep`/`_cine`)이 렌더 경로 전반에 흩어져 20곳에서 분기. 어떤 함수가 어떤 모드에 의존하는지 시그니처로 알 수 없다.
- **측정**: `_golden` 20회 · 모듈 레벨 `let` 91개 중 렌더 모드 관련 다수.
- **패턴**: **Parameter Object** + **Strategy**. 플래그 인자(flag argument) 냄새와 **시간적 결합(temporal coupling)** 제거.
- **처방**: `renderFrame(ctx)` 형태로 `{ dt, t, mode:'live'|'golden'|'cine', seed, hideDynamic }`를 명시 전달. 골든에서 숨기는 대상(배회 엔티티·낙하 파티클·반사 오버레이·먼지)을 `ctx.hideDynamic` 목록 **하나**로 모아 선언적으로 다룬다.
- **검증**: 골든 17/17이 **연속 2회** 그린 + 현재 미해결인 `lodge` 크로스프로세스 플레이키가 재현되는지 확인. 이 리팩터는 그 버그의 **근원 치료 후보**다 — 지금은 "무엇이 골든에서 숨겨지는가"가 코드 전역에 흩어져 있어 누락을 눈으로 찾을 수 없다.
- **위험**: 렌더 핫패스라 인자 전달 오버헤드 주의(객체 재사용으로 할당 회피). 골든 재고정 필요.
- **왜 P1인가**: 유일하게 **미해결 버그(#212 계열)와 직결**된 항목이다. 나머지는 가독성·확장성이지만 이건 정확성이다.

### P2. 매직 넘버 잔존 → 밸런스 데이터 외부화 완주
- **증상**: 임계·배수가 코드 리터럴로 남아 있음. `core/comfort.js`의 `hunger < 25`는 `gaugeSev`의 25와 **같은 개념인데 별개 리터럴**이라 한쪽만 바뀌면 조용히 갈라진다.
- **측정**: `BAL.` 209 vs 하드코드 게이트 후보 25건(베이스라인 고정 중).
- **패턴**: **Named Constant / Configuration Externalization**. 디렉터 방향(수치는 편집 가능한 JSON)과 일치 — 파일 분리보다 **수치 외부화가 주목적**.
- **처방**: 후보 25건을 `data/balance.js`(BAL)로 승격하고, 게이지 임계는 `BAL.gauges.sev = { crit:25, warn:50 }` 단일 출처로. `gaugeSev`와 `comfort.js`가 같은 상수를 읽게 한다.
- **검증**: `node tools/check-hardcode.mjs` 후보 25 → 감소(베이스라인 재고정) + 배터리 102/102 + 밸런스 시뮬 diff-0(리팩터이므로 수치 불변).
- **위험**: BAL 값을 잘못 옮기면 밸런스가 조용히 바뀐다 → **시뮬 diff-0을 반드시 확인**(RELEASE-CYCLE #3).

### P3. God Object 해체 — 긴 함수부터
- **증상**: game.js가 전체의 44%. 최장 `processDay` 488줄은 하루 정산의 모든 규칙(자원·날씨·프로젝트·인카운터·노트)이 한 함수에 눌려 있다.
- **측정**: 11,701줄 / 422함수 / 최장 488줄.
- **패턴**: **Extract Method → Pipeline**. `processDay`는 순서 있는 단계의 나열이므로 `const DAY_STEPS = [decayGauges, tickWeather, runProjects, rollEncounter, …]` 형태의 **파이프라인**이 자연스럽다(각 단계는 `core/`의 순수 함수, 입력=state 스냅샷, 출력=변경 기술).
- **처방**: 함수 하나씩. 우선순위 `processDay`(488) → `resolveExpedition`(429) → `openCraftModal`(444, UI라 별개 축). 이미 `core/` 14모듈이 있으니 **새 구조를 만드는 게 아니라 기존 구조로 옮기는 일**이다.
- **검증**: 각 추출마다 배터리 102/102 + 골든 무회귀 + (정산 관련이면) 시뮬 diff-0. #73이 써온 방식대로 **추출 전후 해시 동일** 확인이 가장 강한 증거.
- **위험**: 한 번에 크게 하면 되돌리기 어렵다 — 함수 1개 = 커밋 1개.

### P4. 전역 가변 상태 → 쓰기 통로 지정
- **증상**: game.js에서 `state.`를 1,002회 직접 접근. 읽기·쓰기가 구분되지 않아 "누가 이 값을 바꿨나"를 정적으로 추적할 수 없고, 그래서 스냅샷 테스트(골든·모달 DOM)에 과의존한다.
- **패턴**: **Command / Encapsulate Field**. 전면 캡슐화는 과하다 — **쓰기만** 통로를 지정한다(읽기는 자유).
- **처방**: 이미 존재하는 `core/economy.js`의 `resAdd/resConsume` 같은 변경 함수를 표준으로 삼고, 자원·게이지·플래그의 **직접 대입을 점진적으로 그 함수들로** 옮긴다. 신규 코드는 처음부터 통로만 쓴다(규약화).
- **검증**: 배터리 + 세이브 왕복(RELEASE-CYCLE #7).
- **위험**: 전면 착수 금지. **신규 코드 규약 + 손대는 김에**가 현실적이다.

### P5. 문자열 UI 조립 → 뷰 빌더 seam
- **증상**: `innerHTML` 50곳에서 HTML을 문자열로 조립. 그래서 모달 DOM 골든이라는 특이한 테스트가 필요해졌고, 이번 글리프 전환에서도 `<img>`→`<span>` 치환이 문자열 정규식 작업이 됐다.
- **패턴**: **Builder / Component seam**. 프레임워크 도입이 아니라 **작은 헬퍼**로 충분(`el(tag, attrs, children)` 또는 리스트 렌더러 1개).
- **처방**: 반복 구조(자원 칩·제작 행·리스트 행)부터 빌더로. 전면 전환은 목표 아님.
- **검증**: 모달 DOM 골든 무변경(마크업 동일 산출).
- **위험**: 이득 대비 범위가 커지기 쉽다 — **반복 3회 이상인 구조만** 대상.

## 4. 착수 순서와 브랜치 전략

1. **P1(렌더 컨텍스트)** — 유일한 정확성 항목이자 미해결 플레이키의 근원 치료. 별도 브랜치 `refactor-render-ctx`.
2. **P2(매직 넘버)** — 저위험·고가치, 디렉터 방향과 일치. P1과 병행 가능.
3. **P3(긴 함수)** — 함수 1개 = 커밋 1개로 상시 진행(#73 트랙 계승).
4. **P4·P5** — 규약으로 두고 "손대는 김에" 원칙. 전면 착수 금지.

**브랜치 규칙**: 리팩터는 UI/기능 브랜치와 **섞지 않는다**. 무손실 증명(게이트 동일)이 리팩터의 유일한 합격 기준인데, 기능 변경이 섞이면 그 증명이 불가능해진다.

## 5. 하지 말 것 (과설계 방지)

- **프레임워크 도입 금지** — React/상태관리 라이브러리는 이 프로젝트의 문제(관문 부재)를 풀지 않고 이식 비용만 만든다.
- **클래스로 감싸기 위한 클래스 금지** — 현 코드의 병목은 "객체가 없어서"가 아니라 **경계가 없어서**다. 필요한 건 모듈 경계·순수 함수·명시적 인자지 상속 계층이 아니다.
- **전면 재작성 금지** — 26,555줄에는 실화면으로만 검증된 암묵 지식(접지 높이·컬링 순서·복셀 피벗)이 박혀 있다. 골든/배터리가 지키는 건 그 지식이다.
- **테스트 없는 리팩터 금지** — 게이트를 끄거나 골든을 무비판 재고정하면서 하는 리팩터는 리팩터가 아니라 회귀다.
