## 운영 규칙
- 검증 실행·판정·커밋은 코디네이터 직접(검증 전담 원칙). 에이전트는 구현+재현 스크립트까지.
- dist를 읽는 검증과 dist를 쓰는 빌드는 동시 실행 금지(직렬).
- 사이클 결과는 릴리즈 보고에 요약(발견 결함 수·수정 여부·잔여 리스크).
- 콘텐츠 배치는 [SCENARIO-FLOWS.md](../design/SCENARIO-FLOWS.md) 동커밋 갱신까지가 완료 정의.
