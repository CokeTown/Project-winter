## 러그·침대 스택 (5)

### [P1] 러그를 겹쳐 깔면 3겹 동심 패널의 상면이 완전 동일 평면이 되어 z-fighting + 하위 러그 층 관통이 발생한다 (디렉터 스크린샷 「러그 3장이 서로 다른 높이」의 직접 재현)

- **위치**:  · 렌즈 3개 중 반증 0
- **증거**: 러그 T3 build(furniture.js:440-445) 실측 — outer B(2.2,0.04,1.5)@y=0.02 → y=[0.000,0.040], x=[-1.1,1.1], z=[-0.75,0.75] / mid B(1.9,0.045,1.2)@y=0.022 → y=[-0.0005,0.0445] / inner B(1.3,0.05,0.7)@y=0.024 → y=[-0.001,0.049]. 한 장이 이미 0.040/0.0445/0.049 3단 계단이다.
  러그의 y는 항상 0으로 못박힌다: rug에는 stackable/surface가 없어(furniture.js:407-412 — surface는 table:138·dresser:173·crate:527 3종뿐) findSupport(game.js:2406-2418)가 절대 러그를 반환하지 않고, moveGhost(game.js:6676)·rotateActive(game.js:7180)가 `item.y = item._support ? item._support.y : 0` → 0. 게다가 collides(game.js:2421)는 `if (DEFS[item.defId].noCollide) return false`로 즉시 반환, 2426행은 상대가 noCollide면 skip → 러그끼리 겹침에 **아무 가드도 고스트 경고도 없다**(항상 초록).
  두 러그를 그리드 1칸(GRID=0.25, game.js:115) 어긋나게 겹치면: A.outer 상면 y=0.040 과 B.outer 상면 y=0.040 이 x=[-0.85,1.1]×z=[-0.75,0.75] = 2.93㎡ 구간에서 **비트 단위로 동일 깊이** → z-fighting(폴리곤 오프셋 전무). 동시에 A.inner(0.049)가 B.outer(0.040) 위로 0.009 솟고, B.mid(0.0445)가 A.outer 위로 0.0045 솟아 러그 위에 직사각형 단이 뜬 것처럼 보인다.
- **재현**: 옥탑방(room 5.6×4.4, shelters.js:44)에서 러그(fp 2.2×1.5)의 배치 가능 중심 범위는 clampToRoom(game.js:2385-2396) 계산상 x∈[-1.64,1.64](폭 3.28)·z∈[-1.39,1.39](폭 2.78)뿐이다. 3장을 안 겹치게 놓으려면 Δx≥2.2·Δz≥1.5의 2+1 격자로 바닥의 86%를 빈틈없이 타일링해야만 하며, 그 외 모든 배치는 반드시 겹친다. 겹쳐도 경고가 없으므로 겹침이 사실상 기본 결과 — 디렉터 스크린샷의 빨강2·베이지1 러그가 서로 다른 높이로 층져 보이는 화면이 정확히 이 상태다(#171 캡슐 씬 = 옥탑+빨간 카펫+베이지 러그).
- **수정안**: ① 러그에 `stackable: true` + `surface: { y: 0.049, w: 2.2, d: 1.5 }`(티어별은 surfaceYByTier: {1:0.065, 2:0.050, 3:0.049})를 부여해 러그 위 러그가 findSupport 경로를 타고 item.y=0.049로 얹히게 한다. ② 또는 러그를 겹침 금지로(noCollide 제거 후 fp만 얇게) 바꿔 겹침 자체를 차단한다. ③ 최소 응급조치로는 러그 3겹 패널의 y 간격(0.0045)이 러그 두께(0.049)보다 작으므로, 같은 좌표계에 러그가 2장 이상일 때 나중 것에 index*0.006 오프셋을 주는 것 — 다만 이는 근본 해결이 아니므로 ①을 권장.

## 바닥 접지 (16)

### [P1] 요트(tugboat)만 실내 바닥 상면이 y=0.09 — 유일한 비-0 바닥이라 그 셸터의 전 가구가 9cm 파묻히고 러그 3티어는 완전히 소실된다

### [P2] rotateActive의 clampToRoom이 지지대만 옮기고 상판 위 소품은 안 옮긴다 — 회전 한 번에 소품이 상판 밖 공중 + 벽 관통

- **위치**:  · 렌즈 3개 중 반증 0
- **증거**: game.js:7175-7178 `item.rot = (item.rot+1)%4;` → `rotateChildren(item, 1);`(옛 중심 item.x/item.z 기준 공전, 7162-7171) → `const [x, z] = clampToRoom(item, item.x, item.z); item.x = x; item.z = z;` — 클램프가 지지대를 옮긴 뒤 자식에게 그 델타를 전파하지 않는다. 대조군: moveGhost(game.js:6672-6682)는 클램프 후 `const dx = x - item.x, dz = z - item.z`를 먼저 구해 `for (const ch of itemsOn(item)) { ch.x += dx; ch.z += dz; }`로 정확히 전파한다 — rotateActive에만 이 보정이 빠졌다. 게다가 회전 실패 되돌리기(7188-7192)는 rot과 rotateChildren(-1)만 되돌리고 7178에서 바뀐 x/z는 복원하지 않으며, rotateChildren(-1)이 **옮겨진** 중심을 기준으로 돌아 자식이 (Δx+Δz, Δz−Δx)만큼 어긋난 자리에 남는다.
- **재현**: 컨테이너(ROOM 6.4×2.9)에서 테이블(fp 1.5×0.9, surface {y:0.77,w:1.3,d:0.72})을 뒷벽에 붙여 z=0.94(rot0 mz 정확값)에 놓고 그 위 양초를 x−0.5 지점에 올린다(findSupport 허용 한계 |dx|≤1.3/2−0.5×0.3=0.5). 회전 1회: rotateChildren(+1)이 양초를 (0, 1.44)로 공전 → clampToRoom이 rot1 mz=0.64로 테이블을 z=0.64로 끌어당김(Δz=−0.30, node 검산) → 자식 미보정. 결과 양초 오프셋 (0, 0.80) > 회전 후 상판 반깊이 0.65 → 상판 밖으로 0.15 나가 y=0.77 공중에 뜬다. 동시에 양초 z=1.44는 자기 방 클램프 mz=1.14를 넘고 뒷벽 내면 z=1.45 코앞이라, fp 0.5인 양초가 z 1.19~1.69를 차지해 벽을 0.24 뚫는다. collides()는 테이블만 보므로 경고 없이 저장된다(7193 scheduleSave). 재로드하면 findSupport 재계산(2609-2615)으로 바닥에 떨어져 '어제 올려둔 양초가 혼자 내려와 있다'가 된다.
- **수정안**: rotateActive에서 클램프 델타를 자식에 전파한다: 7177 앞에 `const px = item.x, pz = item.z;`를 두고 7178 뒤에 moveGhost(6680-6682)와 동일하게 `if (DEFS[item.defId].surface) { const dx = item.x - px, dz = item.z - pz; if (dx || dz) for (const ch of itemsOn(item)) { ch.x += dx; ch.z += dz; syncTransform(ch); } }`. 되돌리기(7188-7192)도 x/z를 px/pz로 복원한 뒤 rotateChildren(-1)을 호출해 원래 중심 기준으로 되돌게 한다.

### [P3] 티어 손질(T1→T2)이 상판 위 소품의 y를 갱신하지 않는다 — 드레서 스택 소품이 0.03 파묻힘(#196 수정의 역방향 이음매)

- **증거**: game.js:9186-9187 `item.tier = next; recolorItem(item, item.colorIdx);` — 티어만 올리고 그룹을 재빌드할 뿐, itemsOn(item)의 y를 다시 계산하지 않는다(recolorItem(game.js:2375-2384)도 자기 그룹만 재생성). 반면 furniture.js:173-174 `surface: { y: 1.13, ... }` + `surfaceYByTier: { 1: 1.10 }`이라 드레서의 스택면은 티어에 따라 1.10 ↔ 1.13으로 변한다. surfaceRectOf(game.js:2403)는 `(DEFS[other.defId].surfaceYByTier || {})[other.tier || 3] ?? s.y`로 그 값을 읽지만, 이 함수는 findSupport(배치·로드 시)에서만 호출된다. 신규 배치는 항상 T1(game.js:6699 `addItem(defId, 0, 0, 0, 0, true, 0, DEFS[defId].tiered ? 1 : 0)`)이므로 이 경로는 정상 플레이의 기본값이다.
